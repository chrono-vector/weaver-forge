# RC2 static blind audit — Batch 1 findings: Release identity & public status

Batch 1 of 4. Static review only; no execution performed. Scope: canonical identity-variable handling, public status wording, post-tag process, and noncanonical-deviation handling for tag `grok-build-witness-v1.0.0-rc2` (`255b357c9ee33c4a9e34b5d9b6e396c53cfe494e`).

| ID | Finding | Detail | Why it is material |
|----|---------|--------|---------------------|
| RB-001 | Silent environment overrides accepted as canonical | `WEAVER_FORGE_URL`, `WEAVER_FORGE_TAG`, `WEAVER_FORGE_COMMIT`, `GROK_BUILD_*`, and `RUST_IMAGE` are read from the environment without any canonical-vs-override distinction or disclosure requirement. A Witness host with any of these pre-set silently reproduces a different target than the fixed rc2 identity while the resulting evidence is still labeled as if it were the canonical run. | Undermines the fixed-identity guarantee that Independent Witness reproduction (C-014) depends on; a Witness could unknowingly audit the wrong revision or image. |
| RB-002 | Stale rc1/rc2 "current package" wording in public status files | Public status/readiness documentation retains language describing an earlier tag as the "current package" after `rc2` was cut, so a reader following only the public status files can be pointed at a superseded identity. | Misdirects a blind Witness to the wrong immutable tag before they ever reach the package itself. |
| RB-003 | Missing post-tag finalization checklist | No checklist exists that a maintainer must complete immediately after cutting an rc tag (confirm local/remote peel match, update status documents, freeze evidence, re-run identity precheck) before any readiness or audit claim is made about that tag. | Leaves an unguarded gap between "tag exists" and "tag is fit to be audited," during which stale or inconsistent claims can be published. |
| RB-004 | No explicit noncanonical deviation mode | The package defines no explicit, clearly labeled "noncanonical" execution mode for hosts that intentionally deviate from the pinned identity (alternate platform, alternate image, alternate commit for local experimentation). Deviating runs are not distinguishable from canonical runs in the resulting evidence. | Evidence produced under any deviation cannot be trusted to represent the canonical rc2 identity unless deviations are self-declaring. |

All four items in this batch are treated as material blockers to a READY verdict. See `INTEGRATED_BLOCKERS.md` for consolidated IDs and `INTEGRATED_REMEDIATION_LIST.md` for mapped remediation steps.
